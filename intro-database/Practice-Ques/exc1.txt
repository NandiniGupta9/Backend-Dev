1. Online Banking System → SQL  
Reason: Requires strong consistency, transactions (ACID), and security.

2. Social Media Platform → NoSQL  
Reason: Handles large, unstructured, rapidly changing data.

3. Real-time Chat Application → NoSQL  
Reason: High speed, real-time messaging, flexible schema.

4. Hospital Patient Records → SQL  
Reason: Data integrity, relationships, and strict schema required.

5. IoT Sensor Data Collection → NoSQL  
Reason: Massive, fast, unstructured data streams.
